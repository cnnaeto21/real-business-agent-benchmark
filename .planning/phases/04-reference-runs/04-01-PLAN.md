Create `scripts/verify-reference.ts`. This script is both the standalone verifier (`npm run verify-reference`) AND called as a function from `run-reference.ts` at the end of orchestration.

**Structure:**
- Export `verifyReference(indexPath: string): Promise<{ pass: boolean; errors: string[] }>` — callable by run-reference.ts
- Top-level `main()` that calls `verifyReference`, prints results, exits 0 on pass / exits 1 on failure
- Wrap in `try { await main(); } catch` pattern (matching validate-schemas.ts)

**`verifyReference` logic:**
1. Read `results/index.json` via `readFile` + `JSON.parse` → type as `IndexEntry[]` (import type from `../src/eval.ts`)
2. Check `index.length !== 9` → push error `Expected 9 entries, found ${index.length}`
3. For each entry: if `!entry.schema_valid` → push error `schema_valid: false for ${entry.harness} × ${entry.model}`
4. Build `EXPECTED_COMBINATIONS` from `HARNESSES.flatMap(h => MODELS.map(m => \`${h}|${m}\`))` — that is all 9 combinations of the 3 harnesses × 3 models
5. Build found Set from `index.map(e => \`${e.harness}|${e.model}\`)` — check each expected combo exists
6. Also check `docs/judge-prompt.md` exists via `access(path, fs.constants.F_OK)` — push error if missing (REF-03)
7. Return `{ pass: errors.length === 0, errors }`

**Constants at top of file:**
```typescript
const HARNESSES = ["inventory-optimization", "pricing-strategy", "financial-forecasting"];
const MODELS = ["anthropic/claude-sonnet-4-6", "openai/gpt-4o", "google/gemini-1.5-pro"];
```

**`main()` output:**
- On pass: print `PASS: All 9 reference runs verified` and each harness×model as `  OK inventory-optimization × anthropic/claude-sonnet-4-6`
- On failure: print each error prefixed with `  FAIL: ` then `process.exit(1)`

Use `import type { IndexEntry } from "../src/eval.ts"` — type-only import, no runtime coupling to eval module.
Use `import { readFile, access, constants } from "fs/promises"` for file ops. Import `join` from `"path"` for path construction. Resolve paths relative to `process.cwd()`.

**`scripts/run-reference.ts` structure:**

Constants at top:
```typescript
const HARNESSES = ["inventory-optimization", "pricing-strategy", "financial-forecasting"];
const MODELS = ["anthropic/claude-sonnet-4-6", "openai/gpt-4o", "google/gemini-1.5-pro"];
const JUDGE_MODEL = "anthropic/claude-sonnet-4-6";
const EST_INPUT = 2500;     // representative subject tokens in
const EST_OUTPUT = 3400;    // representative subject tokens out
const JUDGE_EST_INPUT = 3500;
const JUDGE_EST_OUTPUT = 512;
```

**Step 1: Parse --dry-run flag**
`const DRY_RUN = process.argv.includes("--dry-run");`

**Step 2: --dry-run branch** (exits after printing results, no API calls):
- For each harness: call `loadHarness(harness)` from `"../src/harness.ts"` — print `  PASS harness: ${harness}` on success, catch and print `  FAIL harness: ${harness}: ${err.message}`
- For each model→envVar mapping: check `process.env[envVar]` — print PASS/FAIL accordingly
  - `anthropic/claude-sonnet-4-6` → `ANTHROPIC_API_KEY`
  - `openai/gpt-4o` → `OPENAI_API_KEY`
  - `google/gemini-1.5-pro` → `GOOGLE_API_KEY`
- Print summary, `process.exit(0)`

**Step 3: clearResults()** — delete all UUID directories under `results/`, write `[]` to `results/index.json`
Use `readdir` to list entries, skip `index.json`, `rm({ recursive: true, force: true })` each UUID dir, write `JSON.stringify([], null, 2)` to index.json.

**Step 4: Cost estimation**
Import `calculateCost` from `"../src/cost.ts"`. Compute:
```
for each model:
  for 3 harnesses:
    subjectCost = calculateCost(model, EST_INPUT, EST_OUTPUT)
    judgeCost = calculateCost(JUDGE_MODEL, JUDGE_EST_INPUT, JUDGE_EST_OUTPUT)
    if subjectCost !== -1: totalEstimate += subjectCost
    else: print warning "WARNING: unknown model for cost estimate: ${model}"
    if judgeCost !== -1: totalEstimate += judgeCost
```
Print: `Estimated total cost: ~$${totalEstimate.toFixed(4)} USD (18 API calls: 9 subject + 9 judge)`

**Step 5: Interactive confirmation**
Check `process.stdin.isTTY` — if not TTY (piped/CI), print `Non-TTY stdin detected; pass --dry-run or run interactively` and `process.exit(1)`.
Use `readline.createInterface` to ask `Proceed? (y/N): ` — if answer !== "y", print `Aborted.` and `process.exit(0)`.

**Step 6: runBenchmarkProcess(harness, model): Promise<number>**
Spawn `tsx src/bin.ts --harness ${harness} --model ${model}` using `child_process.spawn`:
```typescript
spawn("tsx", ["src/bin.ts", "--harness", harness, "--model", model], {
  stdio: "inherit",
  env: process.env,
  cwd: process.cwd(),
})
```
Resolve with exit code; handle `"error"` event (spawn failure) → resolve 1.

**Step 7: runWithRetry(harness, model): Promise<"OK" | "RETRY" | "FAILED">**
- First attempt: if exit 0 → return "OK"
- Log `  First attempt failed (exit ${code}), retrying after 5s...` then wait 5 seconds (`await new Promise(r => setTimeout(r, 5000))`)
- Second attempt: if exit 0 → return "RETRY"
- Log `  Both attempts failed for ${harness} × ${model}` → return "FAILED"

**Step 8: Run all 9 combinations sequentially**
Track results: `type RunResult = { harness: string; model: string; status: "OK" | "RETRY" | "FAILED"; compositeScore?: number; costUsd?: number }[]`

For each harness in HARNESSES, for each model in MODELS:
- Print `Running: ${harness} × ${model}`
- Call `runWithRetry(harness, model)`
- If status is "OK" or "RETRY": read the latest entry from `results/index.json` matching this harness+model and extract `composite_score` and `cost_usd` for the summary
- Push result

**Step 9: Print summary table**
Print header + separator + one row per run:
```
harness                   | model                          | status | score | cost USD
---
inventory-optimization    | anthropic/claude-sonnet-4-6   | OK     | 73.33 | $0.057
...
```
Manual string formatting (NOT `console.table` — per research, it truncates columns and is less portable).

**Step 10: Check for any FAILED runs**
If any result has `status === "FAILED"`:
- Print `ERROR: ${n} run(s) failed after retry. Fix failures before committing. No commit attempted.`
- `process.exit(1)`

**Step 11: Call verifyReference**
Import `{ verifyReference }` from `"./verify-reference.ts"`.
Call `verifyReference(join(process.cwd(), "results", "index.json"))`.
If not pass: print each error, print `Verification failed. Do not commit.`, `process.exit(1)`.

**Step 12: Print commit instructions**
On success, print:
```
All 9 reference runs verified. Commit with:
  git add results/ docs/judge-prompt.md
  git commit -m "data(04-reference-runs): add 9 reference run results"
```
(User commits manually — the script does not run git commands.)

Wrap `main()` in try/catch per validate-schemas.ts pattern.
