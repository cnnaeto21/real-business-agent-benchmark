> **Flag name deviation from CONTEXT.md:** CONTEXT.md locked the flag as `--no-eval`. This plan implements `--skip-eval` instead. Reason: Commander v14 treats `--no-*` prefixes as boolean negation flags for an existing `--eval` option (RESEARCH.md Pitfall 1). Since there is no `--eval` flag to negate, `--no-eval` would register as `undefined` rather than `true` when passed. `--skip-eval` is semantically equivalent and avoids this Commander footgun. The camelCase property is `options.skipEval`.
