  <behavior>
    - computeComposite(scores) returns Math.round((a + rt + c) / 3 * 20); all-5 → 100, all-1 → 20, mixed 4/3/5 → 80
    - JudgeResponse Zod schema: requires actionability, reasoning_transparency, completeness each with score (int, min:1, max:5) and rationale (string, min:1)
    - safeParse failure on rawOutput → { schema_valid: false, validation_error: parseResult.error.message, composite_score: 0, scores with all-zero }
    - Judge call uses plain text messages.create (no tools field) and reads response.content.find(b => b.type === "text").text
    - Malformed judge JSON → retry once → if still malformed → throw (caller catches and logs to stderr)
    - writeScored creates results/<run-id>/scored/<model-slug>.json via mkdir+writeFile pattern
    - upsertIndex reads index.json (empty array if missing), filters out existing run_id entry, pushes new entry, writes back
  </behavior>
  <action>
    Create src/eval.ts. Structure:

    1. IMPORTS: Anthropic, z from "zod", readFile/mkdir/writeFile from "fs/promises", join from "path", HarnessSpec from "./types.ts"

    2. TYPES: Define ScoredResult interface (the shape written to scored/<slug>.json) and IndexEntry interface (the shape in index.json). Export both.

    3. JudgeResponse Zod schema:
       ```typescript
       const DimensionScore = z.object({ score: z.number().int().min(1).max(5), rationale: z.string().min(1) });
       export const JudgeResponse = z.object({
         actionability: DimensionScore,
         reasoning_transparency: DimensionScore,
         completeness: DimensionScore,
       });
       export type JudgeResponseType = z.infer<typeof JudgeResponse>;
       ```

    4. computeComposite: `export function computeComposite(scores: JudgeResponseType): number { return Math.round((scores.actionability.score + scores.reasoning_transparency.score + scores.completeness.score) / 3 * 20); }`

    5. callJudge (internal, not exported): async function that takes judgePrompt string, calls Anthropic SDK in plain text mode (model: "claude-sonnet-4-6", max_tokens: 512, temperature: 0), reads textBlock.text, JSON.parse it, safeParse with JudgeResponse schema. If parse fails: retry once (same call). If retry fails: throw Error("Judge returned malformed JSON after retry").

    6. writeScored (exported): mkdir scored dir, writeFile scored result.

    7. upsertIndex (exported): read-modify-write with run_id deduplication. Accepts { indexPath: string, entry: IndexEntry }.

    8. runEval (exported): Main orchestrator.
       - Read judge prompt template from join(process.cwd(), "docs", "judge-prompt.md")
       - Read rubric from join(process.cwd(), "harnesses", harnessName, spec.eval.rubric)
       - Load zodSchema by dynamically importing harnesses/<name>/schema.ts (same pattern as cli.ts loadHarnessSchema)
       - Run zodSchema.safeParse(rawOutput)
       - If safeParse fails: build zero-score ScoredResult (schema_valid: false, validation_error: parseResult.error.message, all scores 0, composite 0), write scored file, upsert index, return
       - If safeParse passes: inject judgePrompt placeholders, call callJudge, compute composite, build full ScoredResult (schema_valid: true), write scored file, upsert index
       - On callJudge failure: catch error, console.error("Judge scoring failed:", err.message), return WITHOUT writing scored file or updating index (raw output already written by caller)
       - Index path: join(process.cwd(), "results", "index.json")

    Commander --skip-eval note: This file does NOT need to know about CLI flags. That integration is in Plan 03 (cli.ts).

    Anti-patterns to avoid (per RESEARCH.md):
    - Do NOT use tool use for the judge call (no tools field, no tool_choice)
    - Do NOT throw on judge failure — catch and log to stderr, exit normally
    - Do NOT append to index.json without filtering by run_id first
    - Do NOT hardcode absolute paths — use join(process.cwd(), ...)
