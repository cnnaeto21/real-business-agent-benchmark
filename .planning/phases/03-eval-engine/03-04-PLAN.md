<objective>
Add a standalone re-eval script that closes the phase goal gap: the eval engine must be able to score **any existing raw output directory** without re-running the LLM, enabling rubric changes to be re-applied to old runs.

Purpose: The ROADMAP goal explicitly calls out this re-application capability as the motivation for decoupling the eval engine from the CLI runner. Currently `runEval` only accepts in-memory `rawOutput` passed from `cli.ts` immediately after a live LLM call. No path exists to re-score a run that already has `results/<run-id>/raw/<model-slug>.json` on disk.

Output: `scripts/re-eval.ts` — a runnable script that reads an existing run from disk, loads the current harness spec (picking up any rubric changes), and calls `runEval` to produce updated scored/ and index.json files.
</objective>

  <action>
Create `scripts/re-eval.ts` as a standalone script (runnable via `tsx scripts/re-eval.ts --run-id <id>`) that reads an existing run from disk and re-scores it using the current harness spec.

**Algorithm:**

1. Parse `--run-id <id>` from `process.argv` (no Commander needed — simple args parsing with a manual check). Exit 1 with usage message if missing: `"Usage: tsx scripts/re-eval.ts --run-id <uuid>"`.

2. Resolve paths:
   ```typescript
   const resultsDir = join(process.cwd(), "results");
   const runDir = join(resultsDir, runId);
   const metaPath = join(runDir, "meta.json");
   const rawDir = join(runDir, "raw");
   ```

3. Read and parse `meta.json` — exit 1 with clear error if file not found:
   ```
   "Error: Run directory not found: results/<run-id>"
   "  Check that the run-id is correct."
   ```
   Extract: `harness` (string), `model` (string), `cost_usd` (number), `latency_ms` (number), `run_date` (string).

4. Derive `modelSlug` from the model string: `model.replace(/\//g, "--")`.

5. Read `results/<run-id>/raw/<model-slug>.json` and JSON.parse it as `rawOutput: unknown`. Exit 1 with clear error if not found:
   ```
   "Error: Raw output not found: results/<run-id>/raw/<model-slug>.json"
   "  Run the benchmark first, or use --skip-eval to generate raw output."
   ```

6. Load the current harness spec using `loadHarness(harness)` from `src/harness.ts`. This picks up any rubric or schema changes since the original run — which is the entire point of this script.

7. Log intent to stdout before calling runEval:
   ```
   Re-scoring run <run-id>
     Harness: <harness>
     Model:   <model>
   ```

8. Call `runEval({ runDir, runId, harnessName: harness, rawOutput, modelSlug, spec, meta: { cost_usd, latency_ms, run_date } })`.
   - `runEval` already handles judge API failures (logs to stderr, exits 0 without writing scored/)
   - `runEval` already handles schema validation failures (writes zero-score result)
   - `runEval` already prints inline scores on success

9. Wrap the entire body in a top-level try/catch:
   ```typescript
   } catch (err) {
     console.error("Re-eval failed:", err instanceof Error ? err.message : err);
     process.exit(1);
   }
   ```

**Imports needed:**
- `import { join } from "path";`
- `import { readFile } from "fs/promises";`
- `import { loadHarness } from "../src/harness.ts";`
- `import { runEval } from "../src/eval.ts";`

**Do NOT:**
- Use Commander (overkill for a single flag in a one-off script)
- Add a new CLI subcommand to bin.ts / cli.ts (CONTEXT.md says "no separate command" — this is a scripts/ utility, not a CLI feature)
- Re-implement any scoring logic (all of it lives in runEval)
- Accept `--harness` or `--model` flags — all metadata comes from meta.json (single source of truth)
